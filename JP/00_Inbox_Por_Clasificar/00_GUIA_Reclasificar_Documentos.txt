3. NOMENCLATURA ESTÁNDAR
================================================================================

Formato:
  YYYYMMDD_Area_TipoDocumento_Tema_Detalle_Idioma_Estado_v##.ext

Campos:
  YYYYMMDD   = Fecha de creación (ej: 20251216)
  Area       = GOV, LEG, ACD, ADM, FIN, RRHH, OPS, TEC, COM, PROY
  Tipo       = Política, Protocolo, Plan, Malla, Unit, Rúbrica, Acta, Contrato, etc.
  Tema       = Tema específico (ej: Admisiones, Convivencia, Math)
  Detalle    = Si necesario, subtema o identificador (ej: G1, PreK, Básica)
  Idioma     = ES, EN, BI (bilingüe)
  Estado     = DRAFT, REVIEW, APPROVED, FINAL
  v##        = v01, v02, v03, etc.
  ext        = .docx, .pdf, .xlsx, .pptx

Ejemplos válidos:
  20251216_GOV_PEI_Documento_Maestro_ES_DRAFT_v01.docx
  20251216_ACD_Malla_Primaria_Math_G3_EN_DRAFT_v01.xlsx
  20251216_FIN_Presupuesto_2026_ES_DRAFT_v01.xlsx
  20251216_RRHH_PerfilDocente_Primaria_EN_APPROVED_v01.docx
  20251216_ADM_Formulario_Inscripcion_ES_APPROVED_v01.pdf

Excepciones (plantillas y recursos):
  TPL_LessonPlan_BI_v01.docx
  Logo_Principal_Color.png
  Branding_PaletaColores.ai

4. ESTADOS DE DOCUMENTOS
================================================================================

DRAFT
  → Usa mientras construyes o editas
  → Incrementa v## por cada cambio mayor
  → Versión editable

REVIEW
  → El documento está en revisión formal
  → Se congela para cambios mayores hasta recibir comentarios
  → Preferir enviar como PDF

APPROVED
  → Aprobado por autoridad correspondiente
  → No cambia contenido sustancial
  → Listo para comunicar

FINAL
  → Versión publicada (sitio web, comunicados oficiales)
  → Solo cambios por erratas menores
